User menyn.
Jag vet att det inte var tvunget till att välja användare men jag tyckte det var rimligt med den funktionen då vi har
en user klass annars är ju den rätt meningslös kände jag så därför har jag tagit med det.
Det gick dock inte helt smärtfritt och jag var nära på att skrota den iden för att från början så hade jag en userlist
som såklart sparade users, men sen skapade jag även income och expense list i user klassen vilket sket sig helt när vi
fick nya lösningen på att läsa in till fil. Så jag sparade till userns egna listor men sen när jag skulle spara users
till user listan så gick ju det bra men där kom även listorna i user klassen med så income och expense sparades på typ
två olika ställen så det blev krock liksom.
Så innan jag gav upp så kom jag på att jag kan ta bort listorna från user klassen, ha en lista i IncomeStorage och
ExpenseStorage som temporära listor som används och håller värden för den aktuellt valda users, så byter man användare
så rensas dessa listor och nya värden från filer som tillhör en user läses in och då funka det bättre.
Sen blev detta en liten extra utmaning för min egen del vilket jag inte hade nått emot.
